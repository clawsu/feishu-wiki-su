# Wiki Summary Report Format

Use this format when generating a summary report after recursively scanning a wiki space.

---

## Output Template

```markdown
# {Space Name} — Wiki Summary Report

**Source:** {wiki URL}
**Generated:** {YYYY-MM-DD HH:MM}
**Total nodes:** {N} ({X} documents / {Y} folders / {Z} other)

---

## Directory Structure

{tree visualization using ├── / └── / │ characters}

---

## Content Summaries

### 1. {Node Title}
- **Type:** {docx / bitable / sheet / folder}
- **Status:** {✅ Complete / 🚧 In progress / ⚠️ Empty}
- **Summary:** {key content in under 100 words}

### 2. {Folder Name}/
- **Type:** folder
- **Children:** {N} nodes
- **Overview:** {what this folder contains}

#### 2.1 {Child Node Title}
- **Type:** docx
- **Status:** ✅ Complete
- **Summary:** {content summary}

---

## Statistics

| Category | Count | Status |
|----------|-------|--------|
| Total | {N} | — |
| Complete | {X} | ✅ |
| In progress | {Y} | 🚧 |
| Empty | {Z} | ⚠️ |
| No permission | {W} | 🔒 |

---

*Generated by feishu-wiki-su*
```

---

## Document Status Rules

| Status | Condition |
|--------|-----------|
| ⚠️ Empty | Word count < 50 or block count ≤ 2 |
| 🚧 In progress | Contains any of: 待补充 / 编写中 / TODO / 🚧 / WIP / Draft |
| ✅ Complete | Everything else |
| 🔒 No permission | API returned 403 |

---

## Summary Extraction Rules

Extract in priority order from the first 50 lines:
1. H2 headings (`## `)
2. Bold phrases (`**text**`)
3. List items (`- ` or `* `)
4. First paragraph longer than 20 characters that is not a heading

Take at most 5 items, join into one sentence. Truncate at 150 characters and append `…`
